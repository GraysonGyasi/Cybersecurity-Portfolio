# Secure SDLC Requirements Analysis

## Project Overview

This project evaluates security requirements throughout the Software Development Life Cycle (SDLC) to ensure applications are designed, developed, tested, and deployed securely.

## Objectives

- Identify security requirements during each SDLC phase
- Evaluate secure development practices
- Recommend security controls
- Improve software security

## Skills Demonstrated

- Secure SDLC
- Application Security
- Cloud Security
- Risk Assessment
- Identity and Access Management
- Security Requirements Analysis

## Tools & Frameworks

- NIST Secure Software Development Framework (SSDF)
- OWASP Top 10
- Microsoft Azure
- Secure Development Practices

## Key Deliverables

- Security Requirements Analysis
- SDLC Security Recommendations
- Risk Assessment
- Compliance Documentation

## Outcome

Developed security requirements and recommendations to strengthen application security throughout the software development lifecycle.
